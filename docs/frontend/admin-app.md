## Admin 管理后台设计（Vue 3 + Element Plus）

本节描述 Beehive Admin 管理后台的导航结构、主要页面以及关键表单/表格设计，用于支持运维与运营场景。

---

### 1. 整体信息架构与导航

建议采用典型中后台布局：左侧侧边栏导航 + 顶部工具栏 + 内容区。

主要一级菜单模块：

- 仪表盘（Dashboard）
- 用户管理（Users）
- 会话与消息（Conversations & Messages）
- 系统配置（Config）
- 运维工具（Ops Tools）

可能的路由结构示意：

- `/admin/login`：后台登录页。
- `/admin`：后台主框架。
  - `/admin/dashboard`
  - `/admin/users`
  - `/admin/users/:userId`
  - `/admin/conversations`
  - `/admin/conversations/:conversationId/messages`
  - `/admin/config`
  - `/admin/ops`

---

### 2. 仪表盘（Dashboard）

**目的**：提供系统运行状态的快速概览。

**主要内容：**

- 卡片统计：
  - 当前在线用户数；
  - 最近 1 分钟/5 分钟的消息 QPS；
  - 错误率（登录失败率、消息发送失败率等）。
- 图表：
  - 在线用户数时间序列曲线；
  - 消息发送/接收 QPS 时间序列；
  - 错误码分布饼图或条形图。

**实现要点：**

- 表层通过 AdminAPIService 暴露汇总接口或从监控系统（Prometheus）聚合数据。
- 前端使用 Element Plus 布局组件配合图表库（如 ECharts）展示。

---

### 3. 用户管理（Users）

#### 3.1 用户列表页

**目标**：支持按条件检索用户并进行基本管理操作。

**主要元素：**

- 筛选表单：
  - 用户 ID / 昵称 / 邮箱；
  - 状态（正常/封禁）；
  - 注册时间区间；
  - 是否在线（由 PresenceService 提供）。
- 列表表格（Element Plus Table）：
  - 列：用户 ID、昵称、邮箱、状态、最近登录时间、是否在线、操作。
- 操作按钮：
  - 查看详情；
  - 封禁/解封；
  - 强制下线（若在线）。

#### 3.2 用户详情页

**内容：**

- 基本信息卡片：
  - 用户 ID、昵称、头像、邮箱、注册时间、状态等。
- 在线状态与 Session：
  - 当前在线设备列表（gatewayId、设备类型、登录时间）；
  - 提供「强制下线」按钮。
- 会话列表：
  - 用户参与的会话/群组列表；
  - 点击可跳转到对应会话详情或消息列表。

**交互：**

- 部分字段（如昵称、头像）支持编辑并保存；
- 对敏感操作（封禁、强制下线）使用弹窗二次确认。

### 4. 会话与消息（Conversations & Messages）

#### 4.1 会话列表页

**目的**：对会话/群组进行检索与基本查看。

**筛选条件：**

- 会话 ID；
- 会话类型（单聊/群聊/频道等）；
- 创建者/成员包含某用户；
- 创建时间区间。

**表格字段：**

- 会话 ID；
- 类型；
- 名称/主题；
- 成员数量；
- 最近活跃时间；
- 操作（查看详情、查看消息）。

#### 4.2 会话详情与成员管理

**内容：**

- 基本信息：名称、类型、创建者、创建时间；
- 成员列表表格：
  - 用户 ID、昵称、角色（群主/管理员/普通成员）、加入时间、状态；
- 操作：
  - 添加成员（弹出选择用户对话框）；
  - 移除成员；
  - 修改成员角色（如提升为管理员）。

#### 4.3 消息查询页

**目的**：按会话和时间区间查询消息，用于客服、审计与排查。

**筛选表单：**

- 会话 ID（从会话列表跳转时自动带入）；
- 发送方用户 ID；
- 时间区间；
- 关键字（针对文本消息）。

**消息列表表格：**

- 时间戳；
- 发送方用户 ID 与昵称；
- 消息简略内容（文本前若干字符）；
- 消息类型（文本/图片/系统通知等）。

**交互：**

- 点击某条记录可在侧边抽屉中查看完整消息详情（包括原始 JSON payload，如有必要）。

### 5. 系统配置（Config）

**目的**：通过可视化界面对系统运行时配置进行管理，配合 etcd 作为配置存储。

**典型配置项：**

- Feature flags（功能开关）：
  - 例如：是否开启某种消息类型、是否开放注册等。
- 限流策略：
  - 登录频率限制；
  - 单用户消息发送 QPS 限制。
- 灰度规则：
  - 按用户 ID 段或标签控制是否加入某个实验或新功能。

**页面形式：**

- 左侧配置分类树（Feature Flags / Rate Limit / Experiments 等）。
- 右侧配置列表：
  - Key、当前值、描述、最后修改人/时间。
- 编辑方式：
  - 行内编辑或点击「编辑」打开对话框修改值；
  - 保存后调用 AdminAPIService，后端更新 etcd 中对应配置键。

---

### 6. 运维工具（Ops Tools）

**功能示例：**

- 消息重放：
  - 输入会话 ID 与时间区间，触发某些消息的重放（仅用于测试环境）。
- 队列检查（RabbitMQ）：
  - 展示关键队列的堆积长度；
  - 提供简单的「清理」或「重启消费」操作入口（仅在有严格权限控制时开放）。
- 服务健康状态：
  - 展示各后端服务（Gateway/Auth/Message/Presence/Conversation 等）的健康检查结果与延迟。

**安全控制：**

- 此模块下的大部分操作应仅对拥有特定角色/权限的管理员可见。
