# IM 客户端界面重设计（参考图规范）

本文档描述按参考图实施的 IM Web 客户端布局与交互规范，与 [client-app.md](client-app.md)、[ux-and-theming.md](ux-and-theming.md) 配合使用。

---

## 1. 整体布局

- **三栏结构**：
  - **左窄导航栏**：固定宽度（可收起为仅图标宽度），顶部为品牌与用户区，下方为功能导航（消息、联系人），最底为设置。
  - **左内容区**：根据当前导航显示「会话列表」或「联系人/群列表」等，带搜索与操作按钮。
  - **右主区域**：当前选中会话的聊天窗口，或联系人详情/空白占位。

- **左侧顶部**：
  - 展示 **Beehive** 品牌名与 **用户头像**（圆形，可点击打开用户菜单：个人资料、退出登录）。
  - **收起左侧边栏时**：仅保留用户头像可见，隐藏 Beehive 文案；展开时恢复 Beehive + 头像。

---

## 2. 左侧导航

- **第一项：消息**
  - 包含群聊与单聊，统一在会话列表中展示。
  - 会话名称：单聊显示对方用户名/昵称，群聊显示群名称。
  - 列表项展示：头像、名称、最后一条消息预览、时间或未读徽章（99+ 封顶）。
  - 群聊可在副标题或角标显示成员数，如「群名称(26)」（数据来自 `conversation.list` 的 `memberCount`）。

- **第二项：联系人**
  - 提供：好友搜索、添加好友入口。
  - 好友通知与群通知入口（红点/数量徽章），点击进入待处理的好友申请、群申请列表。
  - 用户的好友列表与群聊列表（可选项卡切换或分区展示）。

- **最底：设置**
  - 点击进入个人设置页（账号、安全、外观等）。

---

## 3. 群聊窗口

- **顶部标题栏**
  - 显示：**群名称(群人数)**，例如「测试群(26)」。
  - 右侧可放置：语音/视频、添加成员、更多（菜单含「退出群聊」等）。

- **主区域**
  - 消息时间线（与现有 MessagePanel 一致），支持历史加载、已读回执。

- **右侧边栏（群聊时显示）**
  - **群公告**：标题「群公告」，右侧箭头可点击查看详情；若后端支持编辑，仅群主/管理员可编辑。
  - **群聊成员**：标题「群聊成员 N」，带搜索图标；列表展示头像、昵称/用户名，并标注 **群主**、**管理员** 角色（数据来自 `conversation.listMembers` 的 `role`）。

- **单聊窗口**：无右侧边栏或仅显示对方资料卡片；标题为对方昵称/用户名。

---

## 4. 操作与入口

- **好友申请**
  - 发起：联系人页「添加好友」输入用户名/账号，发送申请（调用 `contact.request`）。
  - 通过/拒绝：在「好友通知」列表中对待处理申请执行通过（`contact.accept`）或拒绝（`contact.decline`）。

- **群申请**
  - 申请加群：通过群号或入口发起申请（调用 `group.apply`，若后端支持）。
  - 审批：群主/管理员在「群通知」中对申请执行通过（`group.approve`）或拒绝（`group.decline`）。

- **退出群聊**
  - 群聊窗口更多菜单中「退出群聊」，二次确认后调用 `conversation.removeMember`（`userId` 为当前用户）。

- **删除好友**
  - 联系人列表中某好友操作菜单「删除好友」，二次确认后调用 `contact.remove`。

## 6. 操作与 WebSocket 依赖

| 操作 | 入口位置 | WebSocket 类型 | 后端状态 |
|------|----------|----------------|----------|
| 好友申请（发起） | 联系人页「添加好友」 | `contact.request` | 已实现 |
| 好友申请（通过） | 好友通知列表 | `contact.accept` | 已实现 |
| 好友申请（拒绝） | 好友通知列表 | `contact.decline` | 已实现 |
| 好友申请列表 | 好友通知入口 | `contact.requestList` | 已实现 |
| 群申请（发起） | 群号/入口 | `group.apply` | 已实现 |
| 群申请（审批） | 群通知列表 | `group.approve` / `group.decline` | 已实现 |
| 群申请列表 | 群通知入口 | `group.joinRequestList` | 已实现 |
| 退出群聊 | 群聊窗口更多菜单 | `conversation.removeMember` | 已实现 |
| 删除好友 | 联系人列表好友操作菜单 | `contact.remove` | 已实现 |

实现完成后将上表「后端状态」更新为「已实现」并保持与 [websocket-client-api.md](../API/websocket-client-api.md) 一致。

## 7. 与现有文档关系

- **路由与页面结构**：以 [client-app.md](client-app.md) 为基础，本重设计不改变路由路径（`/app/chats`、`/app/contacts`、`/app/settings`），仅调整 AppShell 左侧布局与 ChatPage 内部结构（群聊增加右侧栏）。
- **UX 与主题**：交互与视觉遵循 [ux-and-theming.md](ux-and-theming.md)（一致性、即时反馈、错误可恢复等）。
- **接口依赖**：群人数依赖 `conversation.list` 返回 `memberCount`；群公告与成员依赖 `conversation.get`、`conversation.listMembers`；好友/群通知依赖 `contact.requestList`、`group.joinRequestList` 等（见 [websocket-client-api.md](../API/websocket-client-api.md)）。操作与 WS 依赖见上文「操作与 WebSocket 依赖」表。
